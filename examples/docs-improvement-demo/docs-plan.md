# Documentation Improvement Plan

## User Confusion

A first-time user understands that the workbench opens multiple panes, but does
not understand what files are created or where the final summary lives.

## Recommended Edits

1. Add an "Example Outputs" section near the top of the README.
2. Link directly to `handoff-summary.md`, `final-triage.md`, and
   `decision-memo.md` examples.
3. Add a short note that all generated files stay in the selected local idea
   folder unless the user exports or publishes them.

## Acceptance Check

A reader should be able to answer, before installing, what the workbench
produces and where those files live.
