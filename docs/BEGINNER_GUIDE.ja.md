# Local Agent Workbench やさしい解説

このページは、Local Agent Workbench を中学生でも分かるように説明するページです。

## このプロジェクトは何か

Local Agent Workbench は、Codex CLI を1つのターミナル画面の中に複数並べて使うための道具です。

普通は、AIに1つずつ質問します。  
でも、難しい開発作業では、いくつかの役割に分けた方が分かりやすくなります。

たとえば:

- 1つ目の画面は、問題の原因を調べる
- 2つ目の画面は、関係するコードを探す
- 3つ目の画面は、テスト方法を考える
- 4つ目の画面は、安全面を確認する
- 5つ目の画面は、最後のまとめを書く

このように、Codex CLI を小さなチームのように使えます。

## 何のために役立つか

- GitHub Issue の原因調査
- Pull Request のレビュー
- リリース前の確認
- 新機能のアイデア出し
- 調査結果を Markdown で残すこと

## たとえ話

学校のグループ発表を考えてください。

1人で全部やるより、調べる人、資料を作る人、発表内容を確認する人に分けた方が進めやすいです。

Local Agent Workbench は、Codex CLI にそのような役割分担をさせるための作業机です。

## 注意点

この道具は、GitHubの画面上でそのまま動くアプリではありません。  
自分のMacに入れて、ローカルで使うツールです。

また、AIが勝手にコードをマージするものではありません。  
最後に確認して判断するのは人間です。
