# ユースケース

## OSSメンテナンス

Issue の再現、関連コード調査、修正案、テスト観点、返信案を分けて進められます。ひとつのチャットで全部を混ぜるより、判断材料が整理されます。

## PRレビュー

pane ごとに、挙動変更、テスト不足、互換性、安全性、ドキュメント影響を分担できます。最後に `close_workflow.sh` でレビュー結果をまとめます。

## リリース準備

CHANGELOG、互換性、スモークテスト、リリースノート、残タスクを並列に確認できます。小さなOSSでも、リリース前の見落としを減らせます。

## 機能発見

ユーザー課題、CLI設計、実装リスク、テスト観点を分けて、作るべき機能を絞り込めます。

## セキュリティ・ドキュメント・依存関係

同梱の examples には、セキュリティ調査、ドキュメント改善、依存関係更新レビューのデモもあります。

- `examples/security-triage-demo`
- `examples/docs-improvement-demo`
- `examples/dependency-update-demo`

## 個人開発者の宣伝材料

単に「AIを使っています」と言うより、導入手順、品質ゲート、サンプル出力、公開リポジトリを示せる方が信頼されやすいです。このプロジェクトは、その実例として見せられます。
