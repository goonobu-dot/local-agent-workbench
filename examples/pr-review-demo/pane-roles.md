Pane 1: Review user-visible behavior and CLI semantics.

Pane 2: Review tests and missing regression cases.

Pane 3: Review safety, path handling, and data-loss risk.

Pane 4: Review backward compatibility and default behavior.

Pane 5: Review release-note impact.

Pane 6: Review documentation quality.

Pane 7: Challenge the leading approval decision.

Pane 8: Suggest a minimal maintainer response.

Pane 9: Synthesize the final PR review.
